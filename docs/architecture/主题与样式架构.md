# 主题与样式架构

## 1. 目标

UEKit Web 的主题系统需要统一 UEKit 本地组件、Element Plus 底层组件、Tailwind CSS 工具类、亮色和深色模式，以及地图和图表等第三方可视化集成。

## 2. 三层模型

```text
设计语义层：--ue-*
  ↓
组件实现层：Vue scoped CSS / Tailwind utilities
  ↓
底层映射层：--el-*、ECharts option、地图 style
```

业务页面应使用“主色、文本、边框、危险、成功”等语义，不直接依赖某个固定十六进制颜色。

## 3. 语义 Token

```css
:root {
  --ue-color-primary: #2563eb;
  --ue-color-background: #f5f7fa;
  --ue-color-surface: #ffffff;
  --ue-color-foreground: #172033;
  --ue-color-border: #dbe3ef;
  --ue-spacing-md: 12px;
  --ue-radius-md: 6px;
}
```

命名表达用途，不表达具体页面。推荐 `--ue-color-danger`，不使用 `--ue-device-card-red`。

## 4. Element Plus 映射

```css
:root {
  --el-color-primary: var(--ue-color-primary);
  --el-bg-color-page: var(--ue-color-background);
  --el-bg-color: var(--ue-color-surface);
  --el-text-color-primary: var(--ue-color-foreground);
  --el-border-color: var(--ue-color-border);
}
```

组件仍可按需使用 Element Plus，但视觉来源应尽量回到 UEKit 语义 Token。

## 5. Tailwind CSS

UEKit 使用 Tailwind CSS v4，主要用于布局、间距、响应式和复杂组合组件。默认关闭 Preflight：

```json
"tailwind": {
  "enabled": true,
  "version": 4,
  "preflight": false
}
```

企业应用通常已经有 Element Plus 和历史业务样式，因此 UEKit 不导入 Tailwind Preflight。需要注意的是，`uekit.css` 仍提供一个受控 Base Layer：它设置 `html/body` 的边距、字体、字号、前景色和背景色，并统一全局 `box-sizing`。这部分属于当前主题契约，接入已有项目时必须审查，而不能把“关闭 Preflight”理解为“没有任何全局样式”。

## 6. scoped CSS 与工具类边界

推荐使用 Tailwind：

- `flex`、`grid`、`gap`、`padding`。
- 断点和响应式布局。
- 简单 hover、focus 和 disabled 状态。

推荐使用组件 CSS：

- Element Plus 深层结构适配。
- 复杂伪元素。
- 需要稳定 BEM 接口的组件。
- 由大量 Token 组合的状态。

## 7. 深色模式

深色模式必须覆盖页面背景、组件表面、文字、边框、Element Plus 浮层、ECharts tooltip 和地图底图及 Marker。

Canvas 和第三方 SDK 不会自动继承 CSS。消费项目需要把主题状态传给 `ChartEcharts.theme` 和 `MapAmap.mapStyle`；组件在相应 Prop 变化时更新配置或重建实例。

## 8. 禁止事项

- 在组件中散落大量十六进制颜色。
- 全量引入 Element Plus 样式作为临时修复。
- 开启 Tailwind Preflight 后不做全站回归。
- 使用 `!important` 作为常规覆盖方式。
- 亮色可用但深色无法阅读。
- 用单一品牌色覆盖所有业务语义。

## 9. 主题变更验收

- Playground 所有组件亮色、深色截图。
- Element Plus 浮层组件实际打开验证。
- 桌面和移动端验证。
- 图表 tooltip 和地图控件验证。
- 消费项目中至少一个真实复杂页面验证。

详细编码要求见[样式与主题规范](../standards/样式与主题规范.md)。
