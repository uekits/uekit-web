# 样式与主题规范

## 1. 基本原则

- 优先使用 UEKit 语义 Token。
- Tailwind CSS 用于布局和状态组合。
- 复杂组件保留局部 CSS。
- Element Plus 通过 CSS Variables 映射。
- 亮色和深色同等重要。

## 2. Token 命名

```text
--ue-color-*
--ue-font-*
--ue-spacing-*
--ue-radius-*
--ue-shadow-*
--ue-size-*
--ue-z-index-*
```

业务语义颜色应单独定义，不把 `success` 滥用于“在线”等不同语义。

## 3. 颜色

- 文本与背景满足可读对比度。
- 危险、警告、成功、信息有明确差异。
- 不用单一蓝色表示所有状态。
- 深色模式不能只做反色。
- 地图 Marker 和图表系列色需要避开底图及背景主色。

## 4. 字体和密度

- 字号使用 Token，不随 viewport 连续缩放。
- 卡片内标题不使用 Hero 级字号。
- 表格、筛选区和工具栏优先紧凑但可点击。
- 字距默认 `0`，不使用负字距压缩中文。

## 5. 间距和尺寸

- 使用统一间距刻度。
- 固定格式区域使用 `minmax`、`aspect-ratio` 或明确约束。
- 动态内容不能导致工具栏或图标按钮跳动。
- 卡片圆角默认不超过 8px，除非组件语义需要圆形。

## 6. Tailwind

- 使用 v4 语法和 `@tailwindcss/vite`。
- Preflight 固定关闭。
- 不在模板中拼接无法被扫描的动态完整类名。
- 重复且有语义的类组合提取为组件，不大量使用 `@apply` 伪造新框架。

## 7. CSS 类

需要公开稳定类名时使用：

```css
.ue-pro-table {
}
.ue-pro-table__toolbar {
}
.ue-pro-table--compact {
}
```

内部纯实现类可由 Tailwind 承担，不必所有节点都有 BEM 类。

## 8. Element Plus 覆盖

优先级：

1. 修改 `--el-*` 变量。
2. 使用 Element Plus 公开 Props。
3. 在本地适配组件中局部覆盖。
4. 最后才使用 `:deep()`。

禁止在全局样式中无作用域覆盖通用 `.el-*` 结构。

## 9. 动画

- 动画服务于状态反馈，不做装饰堆叠。
- 常规反馈 120-240ms。
- 布局变化避免造成页面抖动。
- 加载刷新应区分首次加载和后台静默刷新。

## 10. 验收矩阵

| 维度 | 最低要求                         |
| ---- | -------------------------------- |
| 主题 | 亮色、深色                       |
| 宽度 | 手机、1440 桌面、宽屏            |
| 内容 | 空、正常、超长                   |
| 交互 | Hover、Focus、Disabled、Loading  |
| 浮层 | Dialog、Drawer、Tooltip 实际打开 |
| 集成 | 地图、ECharts 随主题切换         |
