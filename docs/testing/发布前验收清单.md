# 发布前验收清单

## 1. 范围确认

- [ ] 本次是 Registry、CLI 还是两者共同发布。
- [ ] 公开行为变化已经记录。
- [ ] 不相关代码没有混入发布。
- [ ] 规划内容没有被误写成已实现。

## 2. Registry

- [ ] `registry.json` Schema 正确。
- [ ] 新增条目分层正确。
- [ ] npm 和 Registry 依赖完整。
- [ ] 没有循环依赖。
- [ ] 目标路径位于调用方项目内。
- [ ] 条目版本符合 SemVer。
- [ ] 新版本已通过 `pnpm registry:release` 归档，且没有覆盖历史版本。
- [ ] `pnpm registry:build` 通过。
- [ ] `pnpm registry:verify` 通过。
- [ ] `pnpm playground:verify` 通过，12 个条目与 Lock/Base 一致。

## 3. 组件和主题

- [ ] Props、Events、Slots 和导出类型完整。
- [ ] Element Plus 组件及样式按需导入。
- [ ] 没有客户或行业专属数据。
- [ ] 亮色和深色正常。
- [ ] 长文本、空态、加载和禁用态正常。
- [ ] 手机、1440 和宽屏无溢出重叠。
- [ ] 地图、图表等资源正确销毁。

## 4. CLI

- [ ] `init` 在干净项目成功。
- [ ] `add` 能递归安装依赖。
- [ ] 重复执行没有重复配置。
- [ ] 未追踪同名文件受到保护。
- [ ] 本地修改不会被静默覆盖。
- [ ] `diff` 输出正确。
- [ ] `update` 和 `--merge` 已覆盖测试。
- [ ] `--dry-run` 不产生项目副作用。
- [ ] 依赖安装失败不会推进源码、Base 或 Lock。
- [ ] 错误以非零退出码返回。

## 5. 自动化

- [ ] `pnpm security:check` 通过，且未关闭 TLS 证书校验。
- [ ] `pnpm docs:verify` 通过。
- [ ] `pnpm check` 通过。
- [ ] `pnpm test:e2e` 通过。
- [ ] `pnpm smoke:package` 通过。
- [ ] pnpm、npm 以及 Node 20、Node 22 消费链通过。
- [ ] `pnpm licenses:check` 和 Dependency Review 通过。
- [ ] CI 在目标提交通过。

## 6. 文档

- [ ] README 和文档入口可导航。
- [ ] CLI 命令参考已同步。
- [ ] Schema 和配置参考已同步。
- [ ] 新组件有使用说明。
- [ ] 升级或不兼容变化有迁移说明。
- [ ] 没有真实密钥和 Token。

## 7. npm 发布

- [ ] Changeset 类型正确。
- [ ] `@uekits/web` 版本未占用。
- [ ] `npm pack --dry-run` 内容正确。
- [ ] 包含 README、LICENSE、dist 和 bin。
- [ ] npm Trusted Publisher 配置指向正确组织、仓库和 workflow。
- [ ] GitHub `npm` Environment 已启用审批，并限制 `main` 和受保护版本 Tag。
- [ ] 发布 Ref 是 `main` 或与包版本一致的 `v<semver>` Tag。
- [ ] 首发使用 `next` 验证后再决定 `latest`。

## 8. Registry 部署

- [ ] GitHub Pages Source 使用 GitHub Actions。
- [ ] `registry.uekit.com` DNS 和 HTTPS 正常。
- [ ] 部署后 `index.json` 可访问。
- [ ] 至少抽查 `items/button/<version>.json` 和一个 Integration 条目。
- [ ] 部署后验证脚本通过。

## 9. 最终人工复核

- [ ] 在全新目录执行公开文档中的安装命令。
- [ ] 打开 Playground 完成桌面和移动端截图。
- [ ] 比较发布前后组件视觉和交互。
- [ ] 确认回滚提交或上一版本可用。
