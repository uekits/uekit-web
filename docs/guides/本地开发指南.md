# 本地开发指南

本文面向参与 UEKit Web 仓库开发的成员。

## 1. 克隆和安装

```bash
git clone https://github.com/uekits/uekit-web.git
cd uekit-web
corepack enable
pnpm install
```

仓库要求：

- Node.js `>= 22.12`
- pnpm `>= 11.1.3`

请勿在仓库中混用 npm、yarn 生成新的锁文件。

## 2. 启动开发环境

启动 Playground：

```bash
pnpm dev
```

启动本地 Registry 静态服务：

```bash
pnpm registry:build
pnpm dev:registry
```

本地 Registry 默认提供类似地址：

```text
http://127.0.0.1:4174/web/v1/index.json
http://127.0.0.1:4174/web/v1/button.json
```

## 3. 构建 CLI

```bash
pnpm cli:build
```

在仓库中直接执行开发版 CLI：

```bash
pnpm cli -- list
pnpm cli -- view button
pnpm cli -- build
```

`--` 之后的参数会传给 CLI。

## 4. 使用真实 tarball 测试

仅通过 workspace 调用不能覆盖 npm 发布物遗漏文件等问题。发布前必须构建 tarball：

```bash
pnpm package:check
pnpm smoke:package
```

也可以手工执行：

```bash
cd packages/cli
pnpm pack
```

在临时 Vue 项目中使用生成的 `.tgz`：

```bash
pnpm dlx /absolute/path/to/uekits-web-0.1.0.tgz init
pnpm dlx /absolute/path/to/uekits-web-0.1.0.tgz add button avatar
```

如果使用本地 Registry，在临时项目的 `uekit.json` 中把 `registry` 指向本地地址。

## 5. 常用目录

```text
packages/cli/src/          CLI 命令和安装逻辑
registry.json              Registry 清单
registry/                  被分发的标准源码
registry/package.json      私有 Registry 类型验证 Workspace
registry/tsconfig.json     Registry Vue SFC 类型检查入口
registry.schema.json       Registry 协议 Schema
uekit.schema.json          调用方配置 Schema
apps/playground/src/       展示和浏览器验收
apps/registry-server/      Registry 静态站点
scripts/                   构建、检查和发布脚本
tests/                     单元、端到端和冒烟测试
```

不要直接编辑 `apps/registry-server/dist` 等构建产物。应修改 Registry 清单和 `registry/` 源码后重新构建。

## 6. 完整质量检查

```bash
pnpm check
```

该命令覆盖 Registry、类型、Lint、样式、格式、文档和单元测试。浏览器测试单独执行：

其中 Registry 类型检查可以独立运行：

```bash
pnpm --filter @uekits/web-registry-source typecheck
```

VS Code 新安装依赖或调整 `registry/tsconfig.json` 后仍显示旧诊断时，执行 `TypeScript: Restart TS Server` 和 `Vue: Restart Vue Server`。

```bash
pnpm test:e2e
```

发布物冒烟测试：

```bash
pnpm smoke:package
```

三者都通过才可以认为具备发布候选资格。

## 7. 推荐开发循环

```text
修改 registry/ 源码或 CLI
  ↓
registry:build / cli:build
  ↓
Playground 或临时消费项目联调
  ↓
补充 Vitest / Playwright
  ↓
pnpm check
  ↓
pnpm test:e2e
  ↓
pnpm smoke:package
```

## 8. 提交前检查

- Registry 新条目有唯一名称和安全目标路径。
- 依赖已声明，没有隐式依赖。
- 组件同时验证亮色和深色。
- 中文文档已更新。
- 公开行为变更已添加 Changeset。
- 没有提交测试截图、Playwright 报告、临时 tarball 和本地环境文件。
