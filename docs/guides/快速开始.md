# 快速开始

本指南用于在一个已有或新建的 Vue 3 + Vite 项目中接入 UEKit Web。

## 1. 环境要求

- Node.js `>= 20.19`，UEKit Web 仓库开发建议使用 `>= 22.12`
- Vue `>= 3.5`
- Vite
- pnpm、npm、yarn 或 bun 之一

下面以 pnpm 为例。

## 2. 准备 Vue 项目

如果还没有项目：

```bash
pnpm create vite my-admin --template vue-ts
cd my-admin
pnpm install
```

建议先初始化 Git，并提交当前状态。CLI 会写入源码和配置，独立提交方便审查变化。

## 3. 初始化 UEKit Web

```bash
pnpm dlx @uekits/web@latest init
```

本地开发 UEKit Web 时可以使用 tarball，具体见[本地开发指南](./本地开发指南.md)。

初始化会完成：

- 创建 `uekit.json`。
- 创建 `uekit.lock.json`。
- 创建 `.uekit/bases/`。
- 安装 `theme` 条目。
- 配置 Tailwind CSS v4 Vite 插件。
- 在应用入口注入 UEKit 主题样式。
- 安装主题需要的 npm 依赖。

## 4. 按需添加组件

```bash
pnpm dlx @uekits/web@latest add button avatar status-tag
pnpm dlx @uekits/web@latest add search-panel pro-table detail-drawer
```

CLI 会递归安装 Registry 依赖。例如安装高级表格时，不需要手工逐一查找它依赖的基础组件。

## 5. 在页面中使用

组件已经进入本地项目，可以直接通过别名引用：

```vue
<script setup lang="ts">
import { Avatar } from '@/components/ui/avatar';
import { Button } from '@/components/ui/button';
import { StatusTag } from '@/components/ui/status-tag';
</script>

<template>
    <section class="flex items-center gap-3">
        <Avatar name="王小明" />
        <StatusTag label="已启用" tone="success" />
        <Button>编辑用户</Button>
    </section>
</template>
```

以安装文件实际导出的 Props 和事件为准，可用以下命令查看条目：

```bash
pnpm dlx @uekits/web@latest view avatar
pnpm dlx @uekits/web@latest info
```

## 6. 提交哪些文件

以下内容都应该提交到 Git：

```text
uekit.json
uekit.lock.json
.uekit/bases/
src/styles/uekit.css
src/components/
src/layouts/
src/integrations/
```

原因：

- 本地组件是业务项目正式源码。
- Lock 记录安装版本和文件哈希。
- Base 是后续 `diff` 和三方合并所需基准。

## 7. 修改和升级

可以直接修改本地组件。升级前先查看差异：

```bash
pnpm dlx @uekits/web@latest diff button
```

更新未修改组件：

```bash
pnpm dlx @uekits/web@latest update button
```

尝试合并已修改组件：

```bash
pnpm dlx @uekits/web@latest update button --merge
```

如产生冲突，CLI 会保留本地版本并生成 `.uekit-merge` 文件。不要在主分支直接批量升级，推荐在独立分支处理并完成测试。

## 8. 最小验收

```bash
pnpm typecheck
pnpm build
```

同时手工检查：

- 亮色和深色主题。
- Element Plus 组件样式是否正常。
- Tailwind 工具类是否生效。
- 页面是否出现被 Preflight 重置的异常。UEKit 默认关闭 Preflight。

遇到问题请查看[调试与排错指南](./调试与排错指南.md)。
