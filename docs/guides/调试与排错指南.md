# 调试与排错指南

## 1. CLI 找不到 Registry

症状：`list`、`add` 或 `view` 请求失败。

检查：

```bash
curl https://registry.uekit.com/web/v1/index.json
```

本地开发时确认 Registry 服务已经启动，并检查 `uekit.json` 中的 `registry` 地址。地址应指向 `/web/v1`，不要直接写 `index.json`。

## 2. 组件安装后 import 无法解析

检查：

- `uekit.json` 的 aliases 是否与 Vite、tsconfig 一致。
- CLI 写入的目标目录是否存在。
- Registry 文件中的内部别名是否被正确改写。
- `index.ts` 是否导出了组件和类型。

执行：

```bash
pnpm typecheck
pnpm build
```

不要只依赖开发服务器热更新结果。

## 3. Element Plus 组件没有样式

UEKit 不全量导入 Element Plus CSS。检查组件源码是否显式导入对应样式，例如：

```ts
import "element-plus/es/components/button/style/css";
```

同时确认依赖已经安装：

```bash
pnpm why element-plus
```

## 4. Tailwind 工具类不生效

检查：

- `tailwindcss` 与 `@tailwindcss/vite` 是否存在。
- Vite 插件是否已加入配置。
- `src/styles/uekit.css` 是否在应用入口导入。
- 组件文件是否位于 Tailwind 能扫描的源码范围。

UEKit 使用 Tailwind CSS v4，不要直接套用 v3 的 `tailwind.config.js` 教程。

## 5. 页面被全局样式重置

UEKit 默认 `preflight: false`。检查：

- `uekit.json` 是否被手工改成启用 Preflight。
- 项目是否从其他入口导入了 Tailwind Preflight。
- 是否存在重复的 `@import "tailwindcss"`。

Element Plus 项目通常不应在未审计的情况下开启全局 Preflight。

## 6. 深色模式颜色错误

检查：

- 深色模式选择器是否与主题文件一致。
- 组件是否使用语义 Token，而不是硬编码白色或黑色。
- Element Plus `--el-*` 变量是否在深色作用域重新映射。
- ECharts、地图等 canvas/SVG 集成是否监听主题变化并重新设置配置。

## 7. add 不覆盖已有文件

这是保护机制，不一定是错误：

- 未被 Lock 追踪的同名文件默认不覆盖。
- 已安装但本地修改过的文件默认跳过。

先执行：

```bash
pnpm dlx @uekits/web@latest diff button
```

确认可以丢弃本地修改后才使用 `--overwrite`。不要在不理解差异时批量覆盖。

## 8. update --merge 产生冲突

CLI 会保留本地文件，并生成 `.uekit-merge` 文件。处理步骤：

1. 对比当前文件、`.uekit/bases` 和 `.uekit-merge`。
2. 手工合并到正式文件。
3. 删除已处理的 `.uekit-merge`。
4. 执行类型检查和浏览器测试。
5. 核对 Lock 状态。

当前合并属于基础能力，复杂文件重命名和跨文件迁移不能完全自动完成。

## 9. Registry 构建失败

常见原因：

- 条目名称重复。
- `registryDependencies` 指向不存在条目。
- 依赖环。
- `path` 文件不存在。
- 多个文件写入同一个 `target`。
- 路径包含 `..` 或绝对路径。

执行：

```bash
pnpm registry:build
```

根据首个错误修复源清单，不要修改生成产物。

## 10. npm tarball 能构建但无法执行

运行：

```bash
pnpm package:check
pnpm smoke:package
```

重点检查 `packages/cli/package.json` 的 `bin`、`files`、构建入口和 shebang。workspace 内运行成功不能证明 npm 包包含了所有文件。

## 11. Playwright 失败

检查：

- 开发服务器端口是否被其他进程占用。
- 页面是否等待了不稳定网络请求。
- 动画、时间和随机数据是否造成截图抖动。
- 浏览器是否已安装。

```bash
pnpm exec playwright install chromium
pnpm test:e2e
```

## 12. 获取进一步信息

CLI 信息命令：

```bash
pnpm dlx @uekits/web@latest list
pnpm dlx @uekits/web@latest info
pnpm dlx @uekits/web@latest view pro-table
pnpm dlx @uekits/web@latest cat src/components/pro/pro-table/ProTable.vue
```

提交问题时请附上：CLI 版本、Node 版本、包管理器、`uekit.json`（删除敏感地址）、完整错误和最小复现步骤。
